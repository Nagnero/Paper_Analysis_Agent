    프롬프트 편집
    
        
      
    
      
    
        
        
        
        
      
    
      
    
        
    
          
          
        
    
        
    
          
    분석팀은 3단계로 동작합니다: 분석가가 claim을 추출 → 검증가가 원문 근거 확인 → 작가가 한국어 리포트 작성.
    
          
    
            분석가 — claim과 재현가능성 정보 추출 (JSON)
            
    
              
              
    
                
              
    
            
    
          
    
          
    
            검증가 — claim별 원문 근거 검색 (JSON)
            
    
              
              
    
                
              
    
            
    
          
    
          
    
            작가 — 한국어 리포트 작성 (Markdown)
            
    
              
              
    
                
              
    
            
    
          
    
          
    
            핵심 분석 — 표 전용 한국어 핵심 주장 생성 (JSON)
            
    
              
              
    
                
              
    
            
    
          
    
        
    
        
    
          
    각 에이전트별로 백엔드(Claude / Codex)·모델·추론 강도를 지정합니다. 모델은 가운데, 추론 강도는 오른쪽 드롭다운에서 고릅니다. 지원 강도는 모델마다 다릅니다(Opus 4.8: low~max, Sonnet 4.6: max까지·xhigh 없음, Haiku: 강도 미지원, Codex: low~xhigh).
    채팅: 분석 후 사용자가 같은 논문에 대해 질문할 때 사용.

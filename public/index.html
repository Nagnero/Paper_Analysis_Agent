    프롬프트 편집
    
        
      
    
      
    
        
        
      
    
      
    
        
        
        
      
    
      
    
        
        
    
          
    분석팀 오케스트레이터: 사용자의 강조사항을 해석해 분석 지시(directive)와 추가 감사 작업을 만듭니다.
    
          
          
        
    
        
        
    
          
    분석가가 claim을 추출 → 검증가가 원문 근거 확인 → 작가가 한국어 리포트 작성. 핵심 분석은 표를 생성합니다.
    
          
    
            분석가 — claim과 재현가능성 정보 추출 (JSON)
            
    
              
              
    
            
    
          
    
          
    
            검증가 — claim별 원문 근거 검색 (JSON)
            
    
              
              
    
            
    
          
    
          
    
            작가 — 한국어 리포트 작성 (Markdown)
            
    
              
              
    
            
    
          
    
          
    
            핵심 분석 — 표 전용 한국어 핵심 주장 생성 (JSON)
            
    
              
              
    
            
    
          
    
          
    
            근거 탐색 — "이런 주장하는 파트 있어?" 질문에 근거 제시 (싱글, 작성팀 공용)
            
    
              
              
    
            
    
          
    
        
    
        
        
    
          
    에이전트별 백엔드(Claude / Codex)·모델·추론 강도. 가운데가 모델, 오른쪽이 강도(모델마다 지원 범위 다름).
    
          
    
            
            
            
            
          
    
          
    
            
            
            
            
          
    
          
    
            
            
            
            
          
    
          
    
            
            
            
            
          
    
          
    
            
            
            
            
          
    
          
    
            
            
            
            
          
    
          
    
            
            
            
            
          
    
          
    
        
    
        
        
    
          
    작성팀 오케스트레이터: 지시를 본문 / 그림·표 / 인용 / 근거 탐색 모듈로 분류하고 다듬습니다. (수정 작업은 컴파일 모듈이 항상 실행되어 오류 자동 수정 · 근거 탐색은 읽기 전용)
    
          
          
        
    
        
        
    
          
    본문·그림/표 모듈은 계획 → 작성 → 검토 멀티에이전트로 동작합니다(STORM식). 인용은 단일, 컴파일 수정은 변경 시마다 항상 실행.
    
          
    
            범위 지정 (Scope) — 특정 부분만 수정 시 그 줄 범위만 찾아 토큰 절약 (기본 Haiku)
            
    
              
              
    
            
    
          
    
          
    
            계획 (Planner) — 작성 전 개요·계획 수립 (본문·그림 공용)
            
    
              
              
    
            
    
          
    
          
    
            본문 작성 (Writer) — 텍스트 작성·수정 (LaTeX)
            
    
              
              
    
            
    
          
    
          
    
            그림·표 (Writer) — tikz/표 생성·수정 (LaTeX)
            
    
              
              
    
            
    
          
    
          
    
            검토 (Reviewer) — 초안 검토·다듬기 (본문·그림 공용)
            
    
              
              
    
            
    
          
    
          
    
            인용 — .bib 근거 인용 삽입 (LaTeX)
            
    
              
              
    
            
    
          
    
          
    
            컴파일 수정 — 컴파일 오류 자동 수정 (LaTeX)
            
    
              
              
    
            
    
          
    
          
    
            리서치 (웹) — 준 URL/저널 스코프를 읽어 논문 방향 제안 (Claude 웹, 읽기 전용)
            
    
              
              
    
            
    
          
    
        
    
        
        
    
          
    작성팀 에이전트별 백엔드·모델·추론 강도.
